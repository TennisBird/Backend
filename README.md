
# Описание проекта

Приложение с функционалом создания комманд, добавлением задач, отслеживания их статуса, планирования встреч команды. 

## Основные функции

 **Создание и управление задачами**:
   - Возможность создания задач с указанием названия, описания, приоритета, sp, автора.
   - Изменение статуса задач.
   - Назначением наблюдателей и исполнителей.

**Управление командами**:
   - Создание команд и назначение членов команды.
   - Возможность выставления ролей (например, "Разработчик", "Тестировщик").
     
 ## Стек используемых технологий
 - Java
 - Spring Boot
 - Gradle
 - Jpa
 - Log4J
 - Junit, Mockito

## Описание API
## API для управления пользователями
###1. Создание пользователя
- **POST** `/person/`
- **Описание**: Создает нового пользователя.
- **Тело запроса**:
  ```json
  {
  "login": "логин",
  "password": "пароль",
  "first_name": "Имя",
  "last_name": "Фамилия",
  "username": "Имя пользователя",
  "birth_date": "yyyy-MM-dd",
  "mail_address": "email@example.com",
  "telephone_number": "+123456789"
   }
  ```
-   **Ответ**:

    -   **201 Created**: пользователь успешно создан.

   ``` json
   {
  "id": "UUID человека",
  "login": "логин",
  "first_name": "Имя",
  "last_name": "Фамилия",
  "username": "Имя пользователя",
  "birth_date": "yyyy-MM-dd",
  "mail_address": "email@example.com",
  "telephone_number": "+123456789"
   }
   ```
**2. Получение информации о пользователе**

-   **POST** /person/{uuid}

-   **Описание**: Получает информацию о пользователе по его UUID.

-   **Параметры**:

    -   uuid: UUID пользователя.

-   **Ответ**:

    -   **200 OK**: Информация о пользователе.
 ```json
  {
  "id": "UUID человека",
  "login": "логин",
  "first_name": "Имя",
  "last_name": "Фамилия",
  "username": "Имя пользователя",
  "birth_date": "yyyy-MM-dd",
  "mail_address": "email@example.com",
  "telephone_number": "+123456789"
   }
   ```
 -   **204 No Content**: пользователь не найден (возвращается null).
**API для управления задачами**

**3. Обновление информации о человеке**
- **PUT** /person/{uuid}/
- **Описание**: Обновляет информацию о человеке.
- **Параметры**:
   - uuid: UUID человека.
   - username: Новое имя пользователя (необязательный).
   - first_name: Новое имя (необязательный).
   - last_name: Новая фамилия (необязательный).
   - birth_date: Новая дата рождения (необязательный, формат: yyyy-MM-dd).
   - mail_address: Новый email (необязательный).
   - telephone_number: Новый номер телефона (необязательный).
-**Ответ**:
   -**200 OK**: Информация успешно обновлена.
```json
{
  "id": "UUID человека",
  "login": "логин",
  "first_name": "Имя",
  "last_name": "Фамилия",
  "username": "Имя пользователя",
  "birth_date": "yyyy-MM-dd",
  "mail_address": "email@example.com",
  "telephone_number": "+123456789"
}
```
   - **204 No Content**: Человек не найден (возвращается null).

**4. Удаление человека**
- **DELETE** /person/{uuid}
- **Описание**: Удаляет человека по его UUID.
- **Параметры**:
   - uuid: UUID человека.
- **Ответ**:
   - **200 OK**: Человек успешно удалён.
```json
true
```
   - **200 OK**: Человек не найден.
```json
false
```

## API для управления командами
###1. Создание команды
- **POST** /team/
- **Описание**: Создает новую команду.
- **Тело запроса**:
```json
{
  "name": "Название команды"
}
```
**Ответ**:
**201 Created**: Команда успешно создана.
json
```
{
  "id": "ID команды",
  "name": "Название команды"
}
```
###2. Получение информации о команде
**GET** team/{id}
**Описание**: Получает информацию о команде по её ID.
**Параметры**:
   - id: ID команды.
**Ответ**:
   - **200 OK**: Информация о команде.
json
```
{
  "id": "ID команды",
  "name": "Название команды"
}
```
   - **204 No Content**: Команда не найдена (возвращается null).

**Параметры**:
   - id: ID команды.
**Параметры запроса**:
   - name: Новое название команды.
**Ответ**:
**200 OK**: Название команды успешно обновлено.
```json
true
```
   - **204 OK**: Команда не найдена.
```json
false
```
204 No Content: Команда не найдена (возвращается false).
###4. Удаление команды
**DELETE** /team/{id}
**Описание**: Удаляет команду по её ID.
**Параметры**:
   - id: ID команды.
**Ответ**:
200 OK: Команда успешно удалена.
**200 OK**: Название команды успешно удалена.
```json
true
```
   - **204 OK**: Команда не найдена.
```json
false
```

##API для управления задачами
###1. Создание задачи
**POST** /task/
**Описание**: Создает новую задачу.
**Тело запроса**:
```json

{
  "title": "Название задачи",
  "description": "Описание задачи",
  "status": "Статус задачи",
  "priority": "Приоритет задачи",
  "estimate": 5
}
```
**Ответ**:
**201 Created**: Задача успешно создана.
```json

{
  "id": "ID задачи",
  "title": "Название задачи",
  "description": "Описание задачи",
  "status": "Статус задачи",
  "priority": "Приоритет задачи",
  "estimate": 5
}
```
##2. Получение информации о задаче
**GET** /task/{code}
**Описание**: Получает информацию о задаче по её коду.
**Параметры**:
   - code: Код задачи.
**Ответ**:
   - **200 OK**: Информация о задаче.
```json
{
  "id": "ID задачи",
  "title": "Название задачи",
  "description": "Описание задачи",
  "status": "Статус задачи",
  "priority": "Приоритет задачи",
  "estimate": 5
}
```
**204 No Content**: Задача не найдена (возвращается null).

##3. Обновление задачи
**PUT** /task/{code}/
**Описание**: Обновляет информацию о задаче.
**Параметры**:
   - code: Код задачи.
**Параметры запроса**:
   - title: Новое название задачи (необязательный).
   - description: Новое описание задачи (необязательный).
   - status: Новый статус задачи (необязательный).
   - priority: Новый приоритет задачи (необязательный).
   - estimate: Новая оценка задачи (необязательный).
**Ответ**:
   - **200 OK**: Задача успешно обновлена.
```json

   {
     "id": "ID задачи",
     "title": "Название задачи",
     "description": "Описание задачи",
     "status": "Статус задачи",
     "priority": "Приоритет задачи",
     "estimate": 5
   }
```
**204 No Content**: Задача не найдена (возвращается null).

##4. Удаление задачи
**DELETE** /task/{code}
**Описание**: Удаляет задачу по её коду.
**Параметры**:
   - code: Код задачи.
**Ответ**:
   - **200 OK**: Задача успешно удалена.
```json
true
```
**204 No Content**: Задача не найдена
```json
false
```

##5. Назначение работника на задачу
**POST** /task/{code}/role/{role}
**Описание**: Назначает работника на задачу с определенной ролью.
**Параметры**:
   - code: Код задачи.
   - role: Роль работника.
**Параметры запроса**:
   - worker_id: ID работника.
**Ответ**:
**200 OK**: Работник успешно назначен на задачу.
```json
true
```
204 No Content: Работник не найден или не назначен.
```json
false
```

##6. Получение работников с ролью на задаче
**GET** /task/{code}/role/{role}
**Описание**: Получает список работников с определенной ролью на задаче.
**Параметры**:
   - code: Код задачи.
   - role: Роль работников.
**Ответ**:
**200 OK**: Список работников с указанной ролью.

##Взаимодейтсвие клиента и сервера
Клиент может отправлять неавторизованные запросы на:
- /api/auth/register
- /api/auth/login

Авторизованнве запросы на:
- /api/person/
- /api/task/
- etc.

В  основе нашего клиент-серверного взаимодествия лежит JWT , который содержит информацию о пользователе и время дествия его сессии
